# **План проверки и автоматизации мобильного приложения «Мобильный хоспис»**

## 1. Назначение документа

Этот документ описывает стратегию тестирования и подход к автоматизации проверки мобильного приложения «Мобильный хоспис». План определяет объём работ, методы тестирования, критерии покрытия и последовательность выполнения задач.

## 2. Краткое описание приложения

Приложение предоставляет пользователю доступ к информационным материалам хосписа. Основные функции включают:

* просмотр новостей;
* отображение тематических цитат;
* получение информации о самом приложении.

Вход в систему осуществляется по логину и паролю.

**Тестовые данные:**

* Логин: *login2*
* Пароль: *password2*

---

## 3. Цели тестирования

Основные цели:

* подтвердить корректность ключевой функциональности приложения;
* проверить стабильность и удобство использования;
* подготовить фундамент для UI-автоматизации;
* предоставить рекомендации по повышению качества продукта.

---

## 4. Границы тестирования и структура приложения

Тестирование охватывает пользовательские сценарии, доступные после авторизации.

### Навигационные элементы

* боковое меню (кнопка «три полоски»);
* раздел цитат (иконка "бабочка");
* кнопка выхода из профиля (иконка "профиль").

### Пункты меню

* Главная;
* Новости;
* О приложении.

### Разделы приложения

1. **Экран авторизации**

    * поля для логина и пароля;
    * кнопка входа.

2. **Главная страница**

    * карточки новостей;
    * переходы к разделам;
    * кнопка меню.

3. **Новости**

    * просмотр всех новостей;
    * создание и редактирование;
    * удаление и изменение статуса.

4. **Цитаты**

    * отображение списка цитат;
    * свертывание/разворачивание карточек.

5. **О приложении**

    * данные о продукте;
    * переход к пользовательским документам.

---

## 5. Планирование тестирования

### 5.1 Типы тестирования

**Функциональное:**

* проверка пользовательских сценариев;
* проверка UI-элементов;
* тестирование установки и запуска приложения.

**Нефункциональное:**

* юзабилити-оценка;
* совместимость (реальное устройство + эмулятор);
* базовые проверки производительности;
* оценка устойчивости при некорректных вводах.

**По характеру проверки:**

* позитивные тесты;
* негативные тесты;
* исследовательское тестирование.

### 5.2 Техники проектирования тестов

* эквивалентное разбиение;
* анализ граничных значений;
* тестирование переходов между экранами;
* сценарный подход.

## 7. Список проверяемого функционала

### 7.1 Авторизация

* отображение обязательных полей;
* вход с корректными данными;
* сообщение об ошибке при неправильном вводе;
* поведение при пустых полях;
* выход из учётной записи.

### 7.2 Главный экран

* наличие элементов интерфейса;
* переходы к разделам;
* реакция меню;
* взаимодействие с блоком новостей.

* создание записи (валидные/невалидные данные);
* редактирование;
* удаление;
* изменение статуса;
* фильтрация.

### 7.4 Цитаты

* загрузка цитат;
* взаимодействие с блоками (раскрытие/сжатие).

### 7.5 Раздел «О приложении»

* доступность информации;
* переход к документации (политика конфиденциальности/пользовательское соглашение).

## 10. Риски проекта

* отсутствие полной документации приложения;
* невозможность очистки данных через backend;
* часть сценариев может оказаться недоступной для UI-автоматизации;
* нестабильность приложения может усложнить прогон тестов.
